API Documentation

Build on BradPay — all endpoints are available under /api. Authenticate with Firebase Bearer tokens.

Authentication

All API requests require a Firebase ID token sent as a Bearer token in the Authorization header.

Authorization: Bearer <firebase-id-token>

To get a token, sign in via Firebase Auth in your app, then call user.getIdToken().

Endpoints

Wallet

GET/api/me
Get the authenticated user's profile and balance.
GET/api/balance
Get wallet balance in KES (cents).
POST/api/send
Send KES to another user. Body: { "recipient": "uid|email|phone", "amount": 50000, "pin": "1234" } (amount in cents).
GET/api/history?page=1&limit=20
Get transaction history for the authenticated user.
GET/api/lookup?q=email@example.com
Look up a user by email, phone, or UID.

Ledger

GET/api/ledger/status
Get ledger status — block count, chain validity, latest hash.
GET/api/ledger/blocks?limit=10&offset=0
Get paginated blocks from the BradLedger blockchain.
POST/api/ledger/mine
Mine a new block (forces pending transactions into a block).

Trading

GET/api/trade/orderbook?side=buy|sell
Get the order book for a given side.
POST/api/trade/order
Place a buy/sell order. Body: { "type": "buy|sell", "amount": 10000, "price": 105, "pin": "1234" }.
DELETE/api/trade/order/:id
Cancel an open order.

M-PESA (Daraja)

POST/api/daraja/stkpush
Initiate M-PESA STK Push. Body: { "phone": "2547XXXXXXXX", "amount": 50000 } (amount in cents).

System

GET/api/health
System health check — returns status, user count, transaction count.
GET/api/tariffs
Get active fee/tariff schedule.

Data Formats

All amounts are in KES cents (e.g., 50000 = KES 500.00).
Timestamps are ISO 8601 UTC strings.
All responses are JSON.

Rate Limits

Standard rate limits apply: 60 requests per minute per user. Contact us for higher limits.
