BradPay API

Official REST API documentation for developers. Integrate BradPay into your app.

Base URL: https://bradpay.vercel.app

Authentication

Most endpoints require a Firebase Auth token. Include it in the Authorization header:

Authorization: Bearer <firebase-id-token>

Get a token by signing in via Firebase Authentication on the client side, or use a custom authentication flow.

Endpoints

Auth

POST
/api/auth/register
No Auth

Register a new BradPay wallet. Requires Firebase Auth token in header.

Request Body

FieldTypeDescription

phonestringPhone number (254 format)

Response 201

{
  "uid": "abc123...",
  "displayName": "John Doe",
  "phone": "254712345678",
  "balance": 0
}

Auth

M-PESA Daraja

POST
/api/daraja/stkpush
Auth

Initiate an M-PESA STK Push to deposit funds. Sends a payment prompt to the user's phone.

Request Body

FieldTypeDescription
amountintegerrequiredAmount in KES cents (min 100)
phonestringrequiredPhone in 254 format (e.g. 254712345678)

Response 200

{
  "checkout_id": "ws_CO_280620261200000000",
  "message": "STK Push sent. Check your phone to complete payment."
}
POST
/api/daraja/callback
No Auth

Safaricom calls this endpoint after a customer completes (or fails) the STK Push. This endpoint credits the user's balance on success.

Handled automatically — no developer action needed.

POST
/api/daraja/b2c
Auth

Withdraw funds from BradPay to the user's M-PESA account (Business to Customer).

Request Body

FieldTypeDescription

phonestringrequiredPhone in 254 format

Response 200

{
  "conversation_id": "AG_28062026_000000",
  "message": "Withdrawal initiated. Funds will be sent to your M-PESA."
}

BradUSSD

POST
/api/ussd/callback
No Auth

Africa's Talking-compatible USSD callback. The USSD gateway sends user input here and we return the next menu screen.

Request Body

FieldTypeDescription
sessionIdstringUSSD session ID
phoneNumberstringCaller's phone number (e.g. +254712345678)
textstringUser input so far, * separated
serviceCodestringThe USSD code dialed

Response 200

{
  "response": "CON BradPay\n1. Check Balance\n2. Send Money"
}

CON continues the session, END ends it.

Errors

All endpoints return standard HTTP status codes:

  • 200 — Success
  • 201 — Created (resource created)
  • 400 — Bad Request (missing or invalid fields)
  • 401 — Unauthorized (missing/invalid auth token)
  • 404 — Not Found
  • 502 — Bad Gateway (upstream API failure, e.g. M-PESA)

Error responses follow the format:

{
  "error": "Description of what went wrong"
}

Data Formats

  • Amounts: All monetary values are in KES cents (1 KES = 100 cents). Divide by 100 for display.
  • Phone Numbers: International format without leading + (e.g. 254712345678).
  • Timestamps: ISO 8601 format (e.g. 2026-06-28T12:00:00Z).

Rate Limits

Public endpoints are rate-limited to 60 requests per minute per IP. Authenticated endpoints allow 300 requests per minute.
